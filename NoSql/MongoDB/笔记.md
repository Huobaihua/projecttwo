//什么是nosql 泛指非关系型数据库 没有一个统一的架构，两种不同的 NoSQL 数据库之间的差异程度
//nosql包含哪些 键值数据库、列族数据库、文档数据库和图形数据库
//特点是什么（优点）不需要事先设计数据库，可以随时存储自定义的数据格式
键值数据库  扩展性好，灵活性好，大量操作时性能高    数据无结构化，通常只被当做字符串或者二进制数据，只能通过键来查询值
列族数据库  可扩展性强，查找速度快，复杂性低 功能局限，不支持事务的强一致性
文档数据库  数据结构灵活，可以根据 value 构建索引   文件易丢失，限制复杂查询
图形数据库  支持复杂的图形算法  复杂性高，只能支持一定的数据规模
使用弱存储模式技术
没有采用 SQL 技术标准来定义和操作数据库
采用弱事务保证数据可用性及安全性（或根本没有事务处理机制）
主要采用多机分布式处理（DP，Distributed Processing）方式
//mongodb是什么，特点
文档数据库
基于分布式存储的开源的文档数据库系统
面向集合存储，易存储对象类型的数据
支持动态查询
支持完全索引，包含内部对象
支持复制和故障恢复等等

数据库的使用
show dbs 查看所有数据库
db 显示当前数据库
1.数据库的创建
    use 数据库名称  创建数据库(切换数据库)
2.数据库的删除
    db.dropDatabase() 删除当前数据库
3.创建表的时候直接插入数据
    db.表名.insertOne(...) 向该数据库的表中插入数据
    eg: db.aa.insertOne({"name":"xiaolan","age":18,"hobby":["篮球","排球"]})
4.表的创建
    db.createCollection(要创建的集合名称,{可选参数})
    db.createCollection("集合名字") 新增集合
5.表的删除
    db.表名.drop()
6.查看已有的集合
    show collections / show tables 

数据库的增删改
1.增加数据
    insert() 若插入的数据主键已存在则不保存
    db.表名.insert({}) 插入一个数据
    db.表名.insert([{},{}]) 插入多个数据
    
    db.表名.save()  主键_id存在更新数据，不存在插入数据
    db.表名.insertOne()
    db.表名.insertMany({[],[]})
2.查询所有
    db.表名.find()
    db.表名.find().pretty() 改变排列方式
3.修改
    db.表名update()
4.删除
    remove()
    deleteOne() 删除一条数据
    deleteMany() 删除多条数据
    db.表名.deleteMany({}) 删除所有数据
    db.表名.deleteMany({$or[{},{}]}) 删除多个数据
